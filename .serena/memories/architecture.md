# Architecture

## Directory Structure (hardhat-sample/)

```
hardhat-sample/
├── contracts/
│   ├── Counter.sol          # Counter contract (increment/decrement/reset, owner-controlled reset)
│   ├── Counter.t.sol        # Solidity tests using forge-std
│   └── SimpleStorage.sol    # Text + number storage contract
├── helpers/
│   └── getDeployedAddress.ts  # Reads deployed_addresses.json from ignition deployments
├── ignition/
│   ├── modules/
│   │   ├── Counter.ts       # Ignition deploy module
│   │   └── SimpleStorage.ts
│   └── deployments/
│       └── chain-{chainId}/
│           └── deployed_addresses.json  # Auto-generated by ignition deploy
├── tasks/
│   ├── counter.ts           # All Counter Hardhat tasks
│   └── index.ts             # Re-exports all tasks
├── test/
│   ├── Counter.ts           # TypeScript tests (Viem + node:test)
│   └── SimpleStorage.ts
├── hardhat.config.ts        # Hardhat 3 config (defineConfig, plugins, tasks, networks)
├── tsconfig.json
└── package.json
```

## Key Design Patterns

### Deployed Address Resolution
`helpers/getDeployedAddress.ts` reads `ignition/deployments/chain-{chainId}/deployed_addresses.json`.
Tasks call `getDeployedAddress("CounterModule#Counter", chainId)` — no manual address management needed.

### Task Pattern
Each task:
1. Calls `await hre.network.connect()` to get `{ viem }`
2. Gets `chainId` from `publicClient.getChainId()`
3. Resolves contract address via `getDeployedAddress()`
4. Interacts via `viem.getContractAt()`

### Hardhat 3 Config
- `plugins: [hardhatToolboxViem]`
- `tasks: [task1, task2, ...]` (built task objects)
- Networks: `hardhat` (edr-simulated), `coston2/coston/songbird/flare` (http)
